# План тестирования проекта LinguaTune

## 1. Введение

Данный документ описывает подход к тестированию настольного JavaFX-приложения **LinguaTune**,
предназначенного для изучения английского языка по песням.

Цель тестирования — проверить корректность работы основных функций и получить обратную связь
от пользователей по удобству интерфейса.

Проводятся два типа тестирования:

- **Функциональное тестирование** — проверка, что функции работают согласно требованиям.
- **Пользовательское тестирование** — оценка удобства использования интерфейса реальными пользователями.

---

## 2. Область тестирования

В тестирование входят следующие функции:

1. Запуск приложения и загрузка главного окна.
2. Вкладка **«Профиль»**:
    - загрузка имени из `profile.txt`;
    - сохранение нового имени;
    - отображение мини-статистики (количество песен и слов).
3. Вкладка **«Каталог»**:
    - отображение списка песен;
    - поиск по названию и исполнителю;
    - фильтрация по уровню;
    - открытие окна с текстом песни по двойному клику.
4. Вкладка **«Словарь»**:
    - добавление слов в словарь при клике по слову в тексте песни;
    - отображение словаря в таблице;
    - удаление слова;
    - экспорт словаря в CSV-файл.
5. Загрузка базового словаря из `base-dictionary.txt`.

В тестирование **не входят**:

- поддержка других языков интерфейса;
- сетевые функции (онлайн-обновление песен/словаря).

---

## 3. Типы тестирования

### 3.1. Функциональное тестирование

**Цели:**

- убедиться, что все заявленные функции доступны из интерфейса;
- проверить правильность сохранения и загрузки данных (профиль, словарь, файлы);
- проверить работу фильтрации и поиска в каталоге песен.

**Подход:**

- ручное проходение сценариев по заранее подготовленным тест-кейсам;
- проверка на одной рабочей машине под Windows.

### 3.2. Пользовательское тестирование

**Цели:**

- оценить понятность интерфейса для новых пользователей;
- выявить неудобства и места, где пользователи «теряются»;
- собрать предложения по улучшению.

**Подход:**

- проведение коротких сессий (5–10 минут) с одногруппниками;
- использование Google Forms для сбора обратной связи;
- анализ ответов и формулировка выводов.

---

## 4. Среда и инструменты

- ОС: **Windows 10 / 11**
- JDK: **Java 25**
- Сборщик: **Maven**
- Запуск: `mvn clean javafx:run` или запуск `App.main()` в IntelliJ IDEA
- Инструменты:
    - IntelliJ IDEA Community Edition
    - Google Forms для анкет
    - Таблицы в Markdown (файлы в папке `testing/`)

## 5. Тест-кейсы (функциональное тестирование)

### Таблица тест-кейсов

| ID  | Область       | Шаги / входные данные                                                                 | Ожидаемый результат                                                                                           |
|-----|---------------|----------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------|
| TC1 | Запуск        | Запустить приложение через `mvn clean javafx:run` или `App.main()`                   | Главное окно приложения открывается без ошибок. Отображаются вкладки «Профиль», «Каталог», «Словарь».        |

| TC3 | Статистика    | Открыть вкладку «Профиль»                                                             | Число песен = количеству элементов в каталоге. Число слов = количеству строк в словаре.                      |
| TC4 | Каталог: список | Открыть вкладку «Каталог»                                                           | Таблица показывает список доступных песен (минимум 1–3 тестовые песни).                                       |
| TC5 | Каталог: поиск | Ввести в поле поиска часть названия или исполнителя                                  | В таблице остаются только песни, соответствующие поисковому запросу.                                         |
| TC6 | Каталог: фильтр | Выбрать уровень (например, B1) в комбобоксе `Уровень`                               | В таблице отображаются только песни выбранного уровня.                                                       |
| TC7 | Открытие текста | Дважды кликнуть по строке с песней в таблице                                        | Открывается отдельное окно с текстом песни и заголовком (название, исполнитель, уровень).                    |
| TC8 | Клик по слову | В окне текста песни кликнуть по английскому слову, которое есть в базе              | Появляется диалог с переводом и вопросом «Добавить слово в словарь?».                                        |
| TC9 | Добавление слова | В диалоге нажать OK                                                                | Слово появляется во вкладке «Словарь» в таблице слов. Статистика «Всего слов в словаре» увеличивается на 1.  |

| TC11| Экспорт CSV     | Во вкладке «Словарь» нажать «Скачать словарь (CSV)», выбрать файл и папку          | Создаётся CSV-файл. При открытии в Excel русские буквы отображаются корректно (при выборе кодировки 1251).   |
| TC12| Базовый словарь | Кликнуть по слову, которого нет в `base-dictionary.txt`                             | Появляется сообщение, что слово ещё не добавлено в базу, без ошибок и падений.                               |

## 6. Пользовательское тестирование — сценарии

### Сценарий U1 — Новый пользователь

1. Запустить приложение.
2. На вкладке «Профиль» задать своё имя.
3. Перейти во вкладку «Каталог».
4. Найти знакомую песню с помощью поиска.
5. Открыть текст песни.
6. Кликать по словам, посмотреть перевод.
7. Добавить как минимум 2–3 слова в словарь.
8. Перейти во вкладку «Словарь», посмотреть добавленные слова.
9. Экспортировать словарь в CSV.

**Цели:**  
Понять, насколько интерфейс интуитивен для нового пользователя без предварительного объяснения.

### Сценарий U2 — Повторный пользователь

1. Запустить приложение повторно.
2. Проверить, что имя и словарь сохранились.
3. Удалить одно из слов.
4. Ещё раз экспортировать словарь.

**Цели:**  
Проверить поведение приложения при повторном использовании и работу с данными пользователя.

Результаты сценариев и опроса по Google Forms фиксируются в файлах
`testing/test_report.md`, `testing/feedback_summary.md` и `testing/issues.md`.
