# Структура документации проекта LinguaTune

Проект **LinguaTune** — настольное JavaFX-приложение для изучения английского языка по песням.  
Документация по проекту разделена на три основные группы: пользовательская, техническая и управленческая.

---

## 1. Пользовательская документация

### 1.1. README.md

Краткий входной документ для пользователя и разработчика:

- назначение приложения;
- основные возможности;
- требования к окружению;
- установка и запуск (Maven, IntelliJ IDEA);
- краткий обзор интерфейса (Профиль, Каталог, Словарь);
- информация об авторе и контакты.

### 1.2. Руководство пользователя (User Guide)

Отдельный документ (может быть `docs/user-guide.md`), описывающий работу приложения с точки зрения конечного пользователя:

- обзор главного окна и вкладок:
    - **Профиль** — работа с именем пользователя и статистикой;
    - **Каталог** — поиск и фильтрация песен, открытие текста;
    - **Словарь** — просмотр, удаление слов, экспорт в CSV;
- пошаговые сценарии:
    - задать имя пользователя;
    - выбрать песню и открыть текст;
    - кликнуть по слову и добавить его в словарь;
    - выгрузить словарь в CSV;
- раздел «Часто задаваемые вопросы» (FAQ) и типичные проблемы.

### 1.3. Краткая памятка (Quick Start)

Короткий документ (например, `docs/quick-start.md`) на 1–2 страницы:

- минимальные требования;
- последовательность действий «с нуля»:
    1. установить JDK/Maven;
    2. клонировать репозиторий;
    3. запустить `mvn clean javafx:run`;
    4. выбрать песню и кликать по словам;
- мини-схема интерфейса с подписями (по желанию).

## 2. Техническая документация

### 2.1. Архитектурное описание

Документ `docs/architecture.md`:

- общая схема приложения:
    - слой **model**: `Song`, `Playlist`, `Word`;
    - слой **service**: `SongRepository`;
    - слой **controller**: `MainController`;
    - слой **view**: `main-view.fxml`, `style.css`;
- поток данных:
    - загрузка тестовых песен в `Playlist`;
    - отображение в таблице;
    - открытие окна с текстом песни;
    - обработка клика по слову и работа со словарём;
- привязка к файлам ресурсов:
    - FXML, CSS, словари в формате `.txt`.

### 2.2. Описание моделей данных

Документ `docs/data-models.md`:

- класс `Song`:
    - `id` — идентификатор;
    - `title` — название песни;
    - `artist` — исполнитель;
    - `genre` — жанр;
    - `level` — уровень сложности (A1–C1);
    - `lyrics` — текст песни;
- класс `Playlist`:
    - `name` — название плейлиста;
    - `songs` — список объектов `Song`;
- класс `Word`:
    - `text` — английское слово;
    - `translation` — перевод на русский язык;
- связь между сущностями:
    - один `Playlist` содержит много `Song`;
    - таблица словаря содержит список `Word`.

### 2.3. Документация по пользовательскому интерфейсу (UI)

Документ `docs/ui-description.md`:

- структура `main-view.fxml`:
    - верхнее меню «Справка»;
    - вкладки `TabPane` — Профиль, Каталог, Словарь;
- описание основных элементов:
    - таблица песен (`TableView<Song>`) и её колонки;
    - поле поиска и фильтр по уровню;
    - таблица слов (`TableView<Word>`);
    - элементы вкладки профиля (имя, статистика, кнопки);
- описание окна текста песни:
    - заголовки с названием и исполнителем;
    - область с кликабельными словами;
    - кнопка закрытия.

### 2.4. Документация по файлам данных

Документ `docs/data-files.md`:

- `resources/com/linguatune/data/base-dictionary.txt`:
    - формат: строка вида `english=перевод`;
    - комментарии начинаются с `#`;
    - кодировка: UTF-8;
- `profile.txt`:
    - содержит одну строку — имя пользователя;
    - путь: корень приложения;
- `user-dictionary.txt`:
    - формат: `english=перевод` по одной паре на строку;
    - создаётся и обновляется автоматически при работе;
- экспортируемый CSV-файл словаря:
    - формат: `english;translation`;
    - первая строка может содержать заголовок;
    - кодировка: Windows-1251 (удобно для Excel под Windows).

### 2.5. Инструкция для разработчика

Документ `docs/developer-guide.md`:

- требования к окружению (версии JDK, Maven);
- запуск через Maven: `mvn clean javafx:run`;
- запуск через IntelliJ IDEA:
    - открытие проекта по `pom.xml`;
    - настройка SDK и JavaFX;
- добавление новых песен:
    - изменение `SongRepository`;
    - структура тестовых данных;
- основные паттерны использования JavaFX в проекте.
